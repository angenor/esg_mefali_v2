# Implementation Plan: F18 — Mémoire Contextuelle LLM

**Branch**: `018-llm-memory-context` | **Date**: 2026-04-29 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/018-llm-memory-context/spec.md`

## Summary

Livrer la mémoire contextuelle consommée par le LLM à chaque tour de conversation : profil entreprise + projets compactés (lecture systématique sans cache), fenêtre des 15 derniers messages, tool `recall_history(query, k=5)` pour le rappel sémantique de l'historique ancien (RAG pgvector cosine, intra-thread). Embeddings calculés à la persistance des messages (déjà en BackgroundTask F13). Index `ivfflat lists=100` sur `chat_message.embedding`. Budget tokens contrôlé (compaction déterministe descriptions → projets → messages).

Approche technique :
- Module Python `app/chat/memory/` avec trois fichiers ciblés (context_builder, compactors, recall_history).
- Réutilisation de `app/embeddings_client.py` (Voyage `voyage-3.5`, dim 1024) déjà testé en F01/F13.
- Réutilisation des repositories F11 (entreprise) et F12 (projets) — pas de cache entre tours.
- Tool `recall_history` enregistré dans `app/orchestrator/tool_registry.py` via `tool(...)`, schéma Pydantic `extra='forbid'`.
- Migration alembic `0013_f18_chat_message_embedding_index.py` ajoutant `CREATE INDEX ... USING ivfflat (embedding vector_cosine_ops) WITH (lists=100)`.

## Technical Context

**Language/Version**: Python 3.11+
**Primary Dependencies**: FastAPI, SQLAlchemy 2.x, Pydantic v2, httpx (client Voyage)
**Storage**: PostgreSQL 16 + extension pgvector (déjà activée via F01)
**Testing**: pytest + pytest-cov ; ruff pour le lint
**Target Platform**: Linux backend service (.venv local en dev)
**Project Type**: Web service (backend FastAPI + frontend Nuxt 4 séparé — F18 backend uniquement)
**Performance Goals**: build_context p95 < 150 ms ; recall_history p95 < 200 ms
**Constraints**: Budget tokens 2 000 par défaut ; embeddings 1024 dims ; jamais bloquer la persistance d'un message si Voyage indisponible
**Scale/Scope**: < 100 000 messages MVP ; 1 000 PME × ~10 msgs/jour cibles
