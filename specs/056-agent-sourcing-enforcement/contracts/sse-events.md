# Contract — SSE Events (F56)

**Status**: Phase 1 (canonical)

## Event 1 — `message_done.payload.sources` (FR-010)

The existing `message_done` event (F55) is enriched with a `sources` array.

### Schema

```typescript
interface MessageDonePayload {
  message_id: string;          // UUID
  thread_id: string;           // UUID
  agent_run_id: string;        // UUID
  text: string;                // final assistant text
  tool_calls: Array<{...}>;    // existing F55
  sources: SourceRef[];        // NEW (F56)
  degraded?: boolean;          // NEW (F56) — true if sourcing fell back
  sourcing_status: "ok" | "retried_ok" | "failed" | null;  // NEW
}

interface SourceRef {
  source_id: string;           // UUID
  title: string;
  publisher: string;
  url: string;                 // canonical_url || url
  page: string | null;
  section: string | null;
  verification_status: "verified" | "outdated";
  version: string | null;
  citation_index: number;      // ≥1, order of first appearance
  spans: Array<[number, number]>;  // char offsets in `text`
}
```

### Producer

`compose_response` node (F53/F56) builds `sources` from the `cite_source` tool calls of the turn :

```python
def aggregate_sources_from_calls(tool_calls, validation_result) -> list[SourceRef]:
    cite_calls = [c for c in tool_calls if c.name == "cite_source"]
    seen = {}
    for idx, call in enumerate(cite_calls, start=1):
        sid = call.arguments.source_id
        if sid not in seen:
            seen[sid] = SourceRef(
                source_id=sid,
                citation_index=idx,
                spans=spans_for_source(sid, validation_result),
                **fetch_source_metadata(sid),  # title, publisher, url, etc.
            )
    return list(seen.values())
```

### Consumer

The frontend chat F41 store (`frontend/app/stores/chat.ts`) attaches `sources` to the message and triggers re-render via `MessageSources.vue`.

## Event 2 — `unsourced_claim` (FR-005, FR US3)

A new SSE event type emitted by `flag_unsourced_handler` (LLM-triggered) and by the auto-flag rollup in permissive mode.

### Schema

```typescript
interface UnsourcedClaimEvent {
  event: "unsourced_claim";
  data: {
    account_id: string;
    thread_id: string | null;
    message_id: string | null;
    agent_run_id: string;
    claim: string;
    reason: string;
    span: [number, number] | null;  // null if rollup or no span info
    unsourced_flag_id: string | null;  // null if dedup-suppressed
    auto: boolean;                  // true if generated by auto-flag (permissive)
  };
}
```

### Producer

```python
# app/agent/handlers/flag_unsourced.py
await ctx.event_bus_publisher(
    ctx.account_id,
    "unsourced_claim",
    {
        "thread_id": str(thread_id) if thread_id else None,
        "message_id": str(message_id) if message_id else None,
        "agent_run_id": str(ctx.agent_run_id),
        "claim": args.claim,
        "reason": args.reason,
        "span": None,  # LLM-triggered tool calls have no span; auto rollup includes span of first claim
        "unsourced_flag_id": str(new_id) if new_id else None,
        "auto": False,
    },
)
```

For auto-flag rollup (permissive mode), `auto=True` and `span` is the first detected claim's span.

### Consumer

Frontend chat F41 listens via the existing F55 SSE bridge (`/v1/agent/runs/{id}/stream`). The event handler appends a yellow warning chip to the message body or to the corresponding span.

## Event 3 — `text_delta` (existing F53/F55, no schema change)

No change. F56 does not enrich `text_delta`. Source spans are emitted post-message via `message_done.sources`.

## SSE bridge integration

```python
# app/agent/sse_bridge.py — add unsourced_claim to known event types
KNOWN_EVENTS = {
    "text_delta",
    "tool_call_start",
    "tool_call_done",
    "frontend_event",
    "message_done",
    "error",
    "unsourced_claim",   # NEW (F56)
}
```

## Backward compatibility

- `message_done.payload.sources` is **optional** (default `[]`). Old frontend clients that ignore the field continue to work.
- `unsourced_claim` is a new event type ; old clients that don't subscribe to it ignore it gracefully.
- `sourcing_status` field on `message_done.payload` is optional ; null indicates `mode='off'` or no validation occurred.

## Streaming semantics

- `text_delta` events arrive token-by-token during LLM streaming.
- `message_done` arrives ONCE at the end of the turn (after validator + retry resolved).
- `unsourced_claim` may arrive ANY TIME during the turn (when `flag_unsourced` is invoked) OR right before `message_done` (when auto-rollup runs in permissive mode).

The frontend MUST be prepared for `unsourced_claim` to arrive before, during, or after the relevant `text_delta` events.
