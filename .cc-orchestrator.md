Boucle principale (jusqu'à `done = 6` ou stop) :

1. **Lire l'état** : `.cc-deps.json` (deps, done, in_progress, blocked).
2. **Calculer ready** : `ready = {f : f ∉ done ∪ in_progress ∪ blocked
   ET deps(f) ⊆ done}`.
3. **Lancer en // jusqu'à `_max_parallel` = 4** : pour chaque feature
   `ready` non encore lancée, démarrer un sous-agent Phase A.
4. **Attendre les rapports** : transitions Phase A → Phase B → E2E →
   commit/push/PR/merge → done.
5. **Boucler** : recalculer `ready` après chaque merge.

  Phase H y touchent.

**En pratique pour F53-F58** : F53 doit être mergée AVANT que F54-F58

### Détection de conflit dur

Si `gh pr merge` échoue pour conflit avec une PR déjà mergée :
1. `git fetch origin && git checkout <branche-pr>`.
2. Lance un sous-agent Task `general-purpose` : "rebase sur origin/main
   et résous les conflits sans casser les tests".
3. Re-push, re-tente `gh pr merge`.
4. Si 2 tentatives échouent → marquer `blocked`, passer à la suivante.

3. Max 5 itérations parent (correction → e2e-runner) sans réduction des
   rouges → marquer `blocked`.

## Stop conditions

Mets une feature en `blocked` (continue les autres) si :

1. Phase A : `ready_for_implement: false` après 2 retries.
2. Phase B : `tests_status: stuck`, `regression`, `zone_conflict`.
3. Phase B' : `e2e_status: env_setup_failed` non résoluble.
4. Boucle correction E2E stagnante (5 itérations sans progrès).
5. `gh pr create` ou `gh pr merge` échoue après 2 retries.
6. Migration Alembic supprimerait des données.

Continue toujours avec les autres features. Stop global uniquement si :
- État git incohérent.
- Toutes les features restantes sont `blocked`.
- API down > 10 min.
